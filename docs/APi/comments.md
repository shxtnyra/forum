# Comments API

## Создать комментарий
**Endpoint:** `POST /v1/posts/{postId}/comments`

**Описание:** Создает новый комментарий к указанному посту. Можно создать как корневой комментарий, так и ответ на существующий.

**Требования:**
- Требуется аутентификация (JWT токен)

**Тело запроса:**
```json
{
  "text": "Текст комментария",
  "postId": 123,
  "parentId": null
}
```

**Поля DTO:**
- `text` (string, required) - текст комментария (не пустой)
- `postId` (long, required) - ID связанного поста
- `parentId` (long, optional) - ID родительского комментария

**Пример ответа (200 OK):**
```json
{
  "id": 456,
  "createdAt": "2025-06-20T14:30:00Z",
  "author": {
    "id": 789,
    "username": "user123",
    "avatarUrl": "https://example.com/avatars/user123.jpg"
  },
  "postId": 123,
  "level": 1,
  "parentId": null
}
```

## Получить комментарии к посту
**Endpoint:** `GET /v1/posts/{postId}/comments`

**Описание:** Возвращает список корневых комментариев к посту в формате `CommentShortDTO`.

**Пример ответа (200 OK):**
```json
[
  {
    "id": 456,
    "createdAt": "2025-06-20T14:30:00Z",
    "author": {
      "id": 789,
      "username": "user123"
    },
    "postId": 123,
    "level": 1,
    "parentId": null
  },
  {
    "id": 457,
    "createdAt": "2025-06-20T15:00:00Z",
    "author": {
      "id": 790,
      "username": "user124"
    },
    "postId": 123,
    "level": 1,
    "parentId": null
  }
]
```

---

## Получить ответы на комментарий
**Endpoint:** `GET /v1/posts/{postId}/comments/{commentId}/replies`

**Описание:** Возвращает список ответов (`CommentShortDTO`) на указанный комментарий.

**Пример ответа (200 OK):**
```json
[
  {
    "id": 458,
    "createdAt": "2025-06-20T16:00:00Z",
    "author": {
      "id": 791,
      "username": "user125"
    },
    "postId": 123,
    "level": 2,
    "parentId": 456
  }
]
```

---

## Получить детали комментария
**Endpoint:** `GET /v1/posts/{postId}/comments/{commentId}`

**Описание:** Возвращает полную информацию о комментарии в формате `CommentDetailsDTO`.

**Пример ответа (200 OK):**
```json
{
  "id": 456,
  "content": "Текст комментария",
  "createdAt": "2025-06-20T14:30:00Z",
  "updatedAt": null,
  "author": {
    "id": 789,
    "username": "user123",
    "avatarUrl": "https://example.com/avatars/user123.jpg"
  },
  "post": {
    "id": 123,
    "title": "Название поста",
    "createdAt": "2025-06-19T10:00:00Z"
  },
  "parent": null
}
```

## Особенности реализации

### Иерархия комментариев
- Поле `level` указывает уровень вложенности:
    - `1` для корневых комментариев
    - `2+` для ответов
- Поле `parentId` ссылается на родительский комментарий

### Временные метки
- `createdAt` - автоматически устанавливается при создании
- `updatedAt` - заполняется при редактировании (null если не редактировался)

## Возможные ошибки
| Код | Причина                                           |
|-----|---------------------------------------------------|
| 400 | Невалидные данные в запросе                       |
| 401 | Неавторизованный доступ                           |
| 403 | Попытка комментирования скрытого/удалённого поста |
| 404 | Пост/комментарий не найден                        |
